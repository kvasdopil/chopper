# UI Specs - Index

Status: Draft (2026-06-15)

This section defines stable UI behavior for the 3D model playground. Feature specs are organized by user-facing surfaces, modes, and tools rather than by source-code modules.

Related documents:

- [Specifications index](../index.md): Parent catalog for normative project behavior. Read this to navigate between UI, domain, project, and testing specs.
- [Model viewer specs](model-viewer/index.md): Overview of the GLB viewer surface and links to focused feature specs. Read this before changing viewer-level behavior.
- [Viewer shell](model-viewer/viewer-shell.md): Loading, restore, viewport, undo, export, texture display, and failure behavior. Read this before changing the global viewer experience.
- [Object workflow](model-viewer/object-workflow.md): Object identity, selection, naming, visibility, multi-selection, and join behavior. Read this before changing object-list or object-selection behavior.
- [Separation mode](model-viewer/separation-mode.md): Linked-face selection, threshold graph behavior, applying separations, and boundary cuts. Read this before changing separation workflows.
- [Edge-loop tool](model-viewer/edge-loop-tool.md): Loose-edge contact spans, cap/extrusion/cylinder generation, paired-boundary edits, and viewport adjustment behavior. Read this before changing loop-generation tools.
